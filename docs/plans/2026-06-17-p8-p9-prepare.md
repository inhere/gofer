## 4. 卡点 / 风险

- 官方 SDK `AddTool[In,Out]` 泛型签名与 v1.6.x API 细节需以 `go doc` 实证为准（已验证 `NewServer/AddTool/StdioTransport/Server.Run/Tool/CallToolResult/Implementation`）。
- MCP 复用 job.Service 时**不复制执行逻辑**；tail_log 复用 store.ReadLogTail（经 Service 暴露读路径，必要时加薄封装）。
- P9 并发：interaction 的建/答经 HTTP handler 在独立 goroutine 改 jobEntry 状态，需复用 entry.mu；wrapper 轮询读 answer。
- 恢复后 in-memory job 丢失 → 交互仅保证 live job；重启恢复不在验收内。
